# Each line defines an axis or "dimension" for that variable.
# There are different ways to specify values for that grid. 
#
# FILE SYNTAX
#   - Lines containing whitespace are ignored. 
#   - The '#' character denotes comments. Everything in a line after 
#        a '#' character is ignored.
#   - Extra whitespace within a line is ignored besides tokenizing words.
#   - See syntax for the three different kinds of grids below. 
#
#
# REQUIRED LINES (once each):
#    RestartFile [name of restartFile, or NEW to start at pre-main sequence]

#    NeedFile [path/to/file]
#    OutputDir [path/to/dir]
#    OutputFile [path/to/file]
#    InlistVariable [filename] [namespace] [variableName] = [value]
# ------------------------------------------------------------------- #

# -- Grid Specifiers ------------------------------------------------ #


# A set of specified values:
# Syntax: GridVariable varName  values val1 val2 val3 val4 (etc.)
#    GridVariable
#       This keyword tells AutoMESA to interpret the rest of the line 
#       as a grid variable.
#    varName (str)
#       The name of the variable
#    "values" (literal text)
#    val1 ... valN (float)
#       The list of values this variable may take. Note that There may 
#       be as many numbers following varName as you want. 

# This gives a variable called "bar" the values 1.618, 2.718, and 3.14159:
#GridVariable  inlist  myNamespace  bar  values  1.618  2.718  3.14159



# A linear range of values with equal spacing:
# Syntax:  GridVariable inlistFile  namespace  varName  linspace  start  stop  count
#    GridVariable
#       This keyword tells AutoMESA to interpret the rest of the line 
#       as a grid variable.
#    inlistFile (str)
#       The name of the inlist file this variable is associated with
#    namespace (str)
#       The name of the namespace in inlistFile that this variable goes in
#    varName (str)
#       The name of the variable
#    "linspace" (literal text)
#       Indicates that thils variable is to have a range of values generated 
#       by the np.linspace() function
#    start (float)
#       The lowest value in the range
#    stop (float)
#       The highest value in the range
#    count (int) 
#       The number of values (including the end points)

GridVariable  inlist_project  controls  x_ctrl(2) values 0 -2.5e-11 -5.0e-11 -7.5e-11 -10e-11
GridVariable  inlist_project  controls  x_ctrl(3) values 0 0.05 0.15 0.25 0.30
GridVariable  inlist_project  star_job  new_omega  values  0 0.00031 0.00030 0.00027

# Syntax:  TiedVariable inlistFile  namespace  varName  rule GridVariable
#   @NOTE: This section has to be after the GridVariable it references.
#    TiedVariable
#       This keyword tells AutoMESA to interpret the rest of the line 
#       as a variable that relies on a primary GridVariable.
#    inlistFile (str)
#       The name of the inlist file this variable is associated with
#    varName (str)
#       The name of the variable
#    nickname (str)
#       The name of the variable that will go into the folder structure name.
#    rule (str)
#       This defines the relationship between the TiedVariable and the GridVariable
#    GridVariable (str)
#       This is the GridVariable that this TiedVariable will rely upon.
#
#TiedVariable  inlist_project  controls  x_ctrl(2)  (((4/10)-initial_mass)/4e8)  initial_mass False

# Syntax:  GridVariable varName  log10space  start stop count
#    GridVariable

#       as a grid variable.
#    varName (str)
#       The name of the variable
#    "log10space" (literal text)
#       Indicates that this variable is to have a range of values generated 
#       by the np.logspace() function

#    stop (float)
#       10^stop is the highest value in the range4/10

#    nickname (str)
#       The new name for the variable that will be represented in filepaths.
